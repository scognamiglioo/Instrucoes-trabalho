ATENÇÃO!

Quando for postar a sua parte de código, abra uma nova branch e então depois uma pull request, na versão final e concreta do seu projeto.

Não altere o SOBRE NÓS, para evitar conflitos.

COMO ADICIONAR UMA NOVA TAG (PÁGINA) AO SITE

Recomendo seguir o padrão na criação de branch, etapas de commits e por fim, do pull-request.

Vocês podem usar qualquer página de tag já disponível para alterar as informações de acordo com a sua tag, isso é:

Altere a tag <title> da página para sua tag

Altere a <h1> da página para sua tag

<section class="content"> corresponde ao início da parte onde as informações aparecem

Lembre-se de adcionar o JS no final do arquivo, assim como os outros tem para a pesquisa funcionar.

MANUALMENTE

  1. Criar uma branch no GitHub

    Vá até o repositório no GitHub.

    No topo, ao lado do seletor de branch (main), clique e digite o nome da nova branch (ex: branch-teste).

    Clique em Create branch: branch-teste from 'main'.

  2. Editar ou adicionar arquivos na branch

    Certifique-se de que está na branch nova (branch-teste).

    Clique em Add file → Create new file.

    Adicione as suas páginas de TAG

    Clique em Commit changes para salvar na branch.

  3. Abrir um Pull Request

    No repositório, o GitHub deve sugerir:

    "Compare & pull request" → clique nesse botão.

    (Se não aparecer, vá em Pull requests → New pull request.)

    Confirme que está comparando branch-teste → main.

    Escreva um título e descrição (ex: "Adiciona página da branch com botão de retorno").

    Clique em Create pull request.
